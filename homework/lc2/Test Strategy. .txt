# Elvira Kravtsova


«Стратегия тестирования e-commerce systems» 
Описание системы:
"e-commerce systems" список функциональностей:
	* Авторизация пользователя
	* Проверка пользователя
	* Получение списка товаров
	* Корзина:
		** проверка содержимого корзины
		** добавление товаров в корзину

Целевая аудитория:
	* продакт овнер
	* администратор
	* маркетолог
	* Конечный пользователь (покупатель)

Цели тестирования производительности:
	* стабильная работа сайта (сервиса) 365 дней в году
	* Надежность данных пользователей
	* стабильная и высокая пропускная способность
	* снижение затрат в дальнейшем на устранение появившихся проблем"

KPI:
	• Время отклика  -3 сек
	• Обработка запроса к базе - до 5 сек
	• Пропускная способность - n* Гб/день
	• Скорость передачи данных  - n* Мб/сек
	• количество просмотров  - 50k день
	• пользовательских сессий - 5к уникальных пользователей 
	• Количество наименований товаров 50к+
	• Использование ресурсов как сервера (ЦП, памяти, диска)  -- базовый сервер провайдера
	• сети (пропускная способность)  -- базовый пропускная способность провайдера

Тестовая среда:
	* Development  ENV для мануального ембедед тестирования  со своей копией БД
	* Staging ENV для проведения перформенс тестирование со своей копией БД
	* Парамерты сети должны быть как на проде"

Используем след. типы тестирования производительности:
	* Stress testing
	* Load testing
	* Capacity testing
	* stability testing
